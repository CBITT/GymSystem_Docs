Use Case 11 - Ban Member (Brief & Casual & Fully Dressed)

Actors: Admin, Gym System

Brief

Admin can ban member from gym system for an indefinite amount of time.

Casual

Admin can view a list of all members and select from the list a particular member that they wish to ban from the gym system.

    System checks with members class if this member is already banned.
    If already banned, prompt "Member is already banned".
    If not already banned, ban member and prompt "Member successfully banned"
When a member is banned they will be unable to login in to the system until they are unbanned. Their account is effectively locked.

Precondition: 1) Member must first be logged in. 2)Member must first have a membership. 3)Member must not already be banned

Goals: Admin is able to ban member from the system

Business Value: 4

Business Risk: 4

Tasks:

  1. Gym System: The Gym system displays a text box and prompts the user to enter the ID of the member they are searching for
  2. Admin: Types in the ID number and runs a search
  3. Gym System: Returns all information about this member
  4. Admin: Admin reads and confirms if this is the member they were looking for
  5. Gym System: Prompts the user if they want to report this user
  6. Admin: Admin can select yes to confirm
  7. Gym System: System increments how many times the member has been reported and confirms that member has been reported

Postcondition: If member is reported 3 times, the member is banned from the system.
