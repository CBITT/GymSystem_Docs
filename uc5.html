Use Case - Buy Membership / Fully dressed Use Case

  • Use case Number 5
  • Use case Risk: 4
  • Use case Business Value: 5
  • Goal in context: The user buys membership
  • Primary Actor: Member
  • Preconditions: User must be logged in. A member does not necessarily own an active membership. User must have a (bank)account set up

    • Main scenario:
    1. The member selects the option to buy membership.
    2. The member selects the membership period.(1 month/3 month/ 6 month/ 12 month)
    3. The user selects the membership type (off peak/full peak).
    4. The user selects sports type / facility. (Swimming pool / Weight Room etc.)
    5. The system displays purchase details.
    6. The member confirms the payment.
    7. The system deducts x amount from the user account.
      8. 0
    8. The system updates the session count for the member.
    9. The system displays confirmation message.
        10. Extensions:
      1. 7.a The system is unable to charge the amount as the member's balance is too low:
        7.a.a The member choose to top up the account (Use case 19)
        7.a.b The member chooses to cancel out from the purchase.
