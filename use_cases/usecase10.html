Use Case 10 - Register Member (brief,casual,Fully Dressed)

Actors: Member, Gym Admin, Gym System

Brief

A person wants to join a gym.

Casual

A person want to join a gym. A person approaches gym administrator with the query. An admin will fill out registration form taking persons informations. An admin will proces the form. User is registered to the system. User recieves user card.

Fully Dressed

Precondition: None

Goals: The user is registerd to the gym system

Business Value: 4

Business Risk: 2

Trigger: A person want to joing a gym

Main Success Scenerio:

  1. Member: Requests to join a gym
  2. Gym Admin: Fill out Registration form
      3. System validates the input
  3. Gym System: Process the form, Adding new user to the system
  4. Gym System: Issues new user card

Frequency of use: once a week
