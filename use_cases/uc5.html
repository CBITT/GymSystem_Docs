Use Case - Buy Membership / Fully dressed Use Case

  • Use case Number 5
  • Use case Risk: 4
  • Use case Business Value: 4
  • Goal in context: The user buys membership
  • Primary Actor: Member
  • Preconditions: User must be logged in. A member does not necessarily own an active membership. User must have a (bank)account set up

    • Main scenario:
    1. The member selects the option to buy membership.
    2. The member selects the membership period.
    3. The member selects membership type.
    4. The system displays purchase details.
    5. The member confirms the payment.
    6. The system deducts x amount from the user account.
    7. The system updates the session count for the member.
    8. The system displays confirmation message.
        9. Extensions:
      • 7.a The system is unable to charge the amount as the member's balance is too low:
        7.a.a The member choose to top up the account (Use case 19)
        7.a.b The member chooses to cancel out from the purchase.
