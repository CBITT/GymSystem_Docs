Use Case - Top up account / Brief Use Case

  • Use case Number 19
  • Use case Risk: 3
  • Use case Business Value: 4
  • Primary Actor: Member
  • Preconditions: User must be logged in. User must have a Bank Account set up

    • User: selects the amount he/she wants to top up with. System: Reads in credit card details. Validates, and confirms transaction. Updates the members account balance.

    (the purpose of this use case is to illustrate what the system would do in a real life environment.)
